# evals/

Deterministic regression suite for this repo's agent, generated by the
agent-eval-scaffold skill.

## Layout

- `datasets/smoke.jsonl`: one case per line, fields `id`, `input`, and
  optional `expected` (a substring that must appear in the result).
- `golden/<case_id>.json`: the committed expected output per case.
- `harness.py`: the only file you should need to edit. `run_case()` calls
  your agent; `normalize()` strips legitimate nondeterminism.
- `conftest.py`: seed pinning and the `--update-golden` flag.

## Running

```bash
pytest evals/                   # compare against goldens, fail on drift
pytest evals/ --update-golden   # rewrite goldens from current output
```

## Adding a case

Append a line to `datasets/smoke.jsonl`, run `pytest evals/ --update-golden`,
inspect the new golden file, and commit both.

## When goldens drift

A failing golden means agent output changed. If the change is intentional,
rerun with `--update-golden` and review the golden diff in code review like
any other change. Never update goldens without reading the diff.

## Determinism rules

LLM-backed agents must run with temperature 0 and a pinned model, or better,
a recorded or stubbed model for CI. If some output fields still vary, strip
them in `normalize()` instead of loosening assertions.
